## 1. 仓库结构说明

```
autopokemon/
├── AutoPokemon.exe          # 打包后的可执行文件（已纳入版本管理）
├── main.py                  # 主入口
├── setup_profiles.py        # 账号配置工具
├── config.py                # 全局配置
├── exceptions.py            # 自定义异常
├── launcher.py              # 启动器逻辑
├── setup_socks.sh           # 目标服务器 SOCKS5 代理配置脚本
├── requirements.txt         # Python 依赖清单
├── core/                    # 核心业务逻辑
├── modules/                 # 功能模块（login_logic、session_runner 等）
├── utils/                   # 工具函数（data_manager 等）
├── data/                    # 运行数据
│   ├── accounts.csv         # 账号列表
│   ├── browsers.csv         # 浏览器/Profile 状态
│   └── emails.csv           # 邮箱列表
├── python/                  # 嵌入式 Python（目标机器手动放置，见下文）
│   └── msvcp140*.dll        # VC++ Runtime DLL（随仓库分发，共 5 个）
├── testpages/               # 测试 HTML 页面
└── .gitignore
```

## 2. 哪些文件被纳入版本管理

### ✅ 已追踪（会随 push 同步到 GitHub）

| 路径 | 说明 |
|------|------|
| `*.py` 源码 | 全部纳入 |
| `AutoPokemon.exe` | 打包产物，随源码一起分发 |
| `setup_socks.sh` | 服务器端脚本 |
| `data/*.csv` | 账号/浏览器/邮箱数据 |
| `python/msvcp140*.dll` | VC++ Runtime（共 5 个），Windows Server 通常缺少 |
| `core/`, `modules/`, `utils/` | 所有子模块源码 |
| `testpages/` | 测试页面 |
| `.gitignore` | 自身 |

### ❌ 已排除（`.gitignore` 中）

| 路径 | 排除原因 |
|------|---------|
| `logs/` | 运行时生成，体积大 |
| `history/` | 旧版本手动备份，已被 Git 历史替代 |
| `venv/`, `.venv/` | 虚拟环境，按 `requirements.txt` 本地重建 |
| `__pycache__/`, `*.py[cod]` | 编译缓存 |
| `python/**`（除 DLL 外） | 嵌入式 Python 体积大，目标机器自行下载 |
| `Nstbrowser*.exe`, `*.zip` | 第三方分发包 |
| `README.md` | 已改为本手册取代 |
| `.env` | 敏感配置 |
| `ossutil.exe`, `OSS操作手册.md` | OSS 工具，不纳入项目 |

## 3. 日常开发工作流

### 3.1 修改代码后推送

```powershell
cd "c:\Users\YY\Desktop\autopokemon"

# 查看哪些文件有改动
git status

# 暂存改动（指定文件）
git add modules/login_logic.py modules/session_runner.py

# 或暂存全部改动
git add .

# 提交（写清楚改了什么）
git commit -m "fix: 修复登录后空白等待问题"

# 推送到 GitHub main 分支
git push origin master:main
```

### 3.2 更新 AutoPokemon.exe 后推送

```powershell
# 重新打包后，直接覆盖 AutoPokemon.exe，然后：
git add AutoPokemon.exe
git commit -m "build: 更新打包版本 vX.X"
git push origin master:main
```

### 3.3 同步远端最新代码（多人协作或多台机器）

```powershell
git pull origin main
```

## 4. Commit 提交规范

使用统一前缀方便追溯：

| 前缀 | 适用场景 | 示例 |
|------|---------|------|
| `fix:` | 修复 bug | `fix: IP 封禁检测逻辑错误` |
| `feat:` | 新功能 | `feat: 添加账号重置错误异常类` |
| `build:` | 打包/构建相关 | `build: 更新 AutoPokemon.exe v1.3` |
| `data:` | 仅更新 CSV 数据 | `data: 更新 browsers.csv` |
| `config:` | 配置变更 | `config: 调整 IP 封禁阈值` |
| `refactor:` | 重构（不改功能） | `refactor: 整理 session_runner 结构` |
| `docs:` | 文档 | `docs: 更新版本管理手册` |

## 5. 部署到目标机器

### 5.1 首次部署（全新机器）

```powershell
# 1. 克隆仓库
git clone https://github.com/chenkeweii/autopokenmon.git autopokemon
cd autopokemon

# 2. 手动配置嵌入式 Python
#    从 https://www.python.org/downloads/windows/ 下载 Python 3.13.1 embeddable package
#    解压全部文件到 autopokemon\python\ 目录
#    （msvcp140*.dll 已经在 python\ 目录中，无需重复复制）

# 3. 直接运行
.\AutoPokemon.exe
```

### 5.2 更新已部署的机器

```powershell
cd autopokemon

# 拉取最新代码和 exe
git pull origin main

# 重启程序即可
```

### 5.3 python\ 目录结构（目标机器配置后）

```
python/
├── python313.dll         ← 来自嵌入包（手动放置）
├── python313.zip         ← 来自嵌入包（手动放置）
├── python.exe            ← 来自嵌入包（手动放置）
├── ... 其他嵌入包文件 ...
├── msvcp140.dll          ← 来自 GitHub（已包含）
├── msvcp140_1.dll        ← 来自 GitHub（已包含）
├── msvcp140_2.dll        ← 来自 GitHub（已包含）
├── msvcp140_atomic_wait.dll  ← 来自 GitHub（已包含）
└── msvcp140_codecvt_ids.dll  ← 来自 GitHub（已包含）
```

> **为什么需要这 5 个 DLL？**  
> Python 的 `greenlet`（Playwright 依赖）在加载时需要 VC++ 2015-2022 Runtime。  
> Windows Server 通常未预装，导致 `DLL load failed while importing _greenlet` 错误。  
> 将 DLL 放在 `python\` 目录旁边可确保加载成功，无需在目标机器上安装 Visual C++ Redistributable。

## 6. 常用 Git 操作速查

### 查看历史

```powershell
# 简洁单行历史
git log --oneline -10

# 含时间和作者
git log --pretty=format:"%h %ad %s" --date=short -10
```

### 查看某次提交的改动内容

```powershell
git show 57acda5
```

### 对比当前改动 vs 上次提交

```powershell
git diff HEAD
```

### 回滚某个文件到上次提交的版本

```powershell
git checkout HEAD -- modules/login_logic.py
```

### 撤销最近一次 commit（保留文件改动）

```powershell
git reset --soft HEAD~1
```

### 彻底回滚到某个历史版本（危险，不可恢复）

```powershell
# 先查到目标 commit hash
git log --oneline

# 回滚（本地工作区也会变）
git reset --hard <commit-hash>

# 强制推送（会覆盖远端历史，谨慎使用）
git push origin master:main --force
```

### 查看远端仓库状态

```powershell
git remote -v
git fetch origin
git log origin/main --oneline -5
```

## 7. 注意事项

1. **data/ CSV 文件含账号信息**，push 前确认内容是否适合上传至公开仓库。如需保密，将 `data/` 重新加入 `.gitignore`。

2. **AutoPokemon.exe 体积较大**，每次打包后都提交会增大仓库体积。建议仅在发布稳定版本时提交 exe，日常只提交源码。

3. **嵌入式 Python 不在仓库中**，克隆后必须手动放置 Python embeddable package，否则 `AutoPokemon.exe` 无法运行。

4. **本地分支 `master` → 远端分支 `main`**，推送命令固定为：
   ```powershell
   git push origin master:main
   ```
   不要直接 `git push`，否则会尝试推送到不存在的远端 `master`。

5. **修改 `.gitignore` 后**，对已被追踪的文件需要手动取消追踪：
   ```powershell
   git rm --cached <文件路径>
   git commit -m "config: 从版本管理中移除 xxx"
   ```
